# Mobile PWA UI change

Implement or refactor a mobile-first PWA UI flow.

Before editing:
- inspect existing terrain shell/navigation/component pattern
- identify primary mobile action
- identify loading, empty, error, offline, unauthorized states

Constraints:
- mobile layout first
- no desktop-only or hover-only UX
- no horizontal scroll
- critical action reachable on phone
- use existing terrain components/patterns
- preserve route/back behavior

Tests:
- route/navigation if changed
- permission visibility if changed
- form/mutation states if changed
- empty/error/offline state if changed

Final:
- UI changed
- Mobile/PWA states covered
- Validated
- Risks
